## 1. ABA DE TREINO — DASHBOARD INTELIGENTE

### 1.1 Dashboard de Calorias do Treino
- Crie um dashboard dentro da aba de treino que exiba:
  - **Média total de calorias gastas** caso o treino seja feito completo.
  - **Quantidade de calorias consumidas** pelo cliente no dia.
  - **Comparativo visual** (gráfico simples) de "calorias gastas no treino" vs "calorias consumidas".
- Os números devem ser claros, grandes e fáceis de entender.

### 1.2 Personalização de Treino por Grupo Muscular
- Crie uma **nova funcionalidade de personalização de treino** (separada de qualquer personalização que já exista no site — esta é uma feature nova).
- O cliente deve poder personalizar seu treino escolhendo exercícios por **grupo muscular**. Os grupos são:
  - Peito
  - Costas
  - Ombros
  - Bíceps
  - Tríceps
  - Pernas (Quadríceps, Posterior, Glúteos, Panturrilha)
  - Abdômen
- **Ao abrir a personalização de cada grupo muscular** (ex: Peito), o sistema deve:
  - Listar **todos os exercícios disponíveis** para aquele grupo.
  - Exibir **no máximo 4 recomendações** do que o cliente deveria fazer, personalizadas conforme o **plano de treino dele**.
  - Permitir que o cliente selecione/desselecione exercícios para montar seu treino.
- Repetir essa lógica para **todos os grupos musculares**.

## 2. CONTABILIZAÇÃO DE CALORIAS — TELA INICIAL + DASHBOARD

### 2.1 Lógica de Contabilização por Foto do Prato
Refatore toda a lógica de contabilização de calorias para funcionar assim:
1. O cliente **tira/envia uma foto do prato**.
2. O sistema identifica o prato e a **quantidade de calorias** (baseado nas fotos e dados de calorias já existentes no site).
3. **Automaticamente soma no total diário** de calorias consumidas.
4. Essa contabilização aparece tanto na **tela inicial** quanto na **aba de dashboard**.

### 2.2 Dashboard Interativo — Calorias Consumidas x Planejado
- Crie um dashboard interativo que mostre:
  - **Calorias consumidas** (total do dia, atualizado em tempo real conforme fotos são adicionadas).
  - **Calorias planejadas/propostas** pelo cliente no planejamento.
  - **Comparativo visual** de pretendido x realizado.

### 2.3 Feedbacks e Alertas Automáticos
- **Refeições que não batem a meta:** exibir feedback claro informando quais refeições ficaram abaixo ou acima.
- **Quantidade de refeições:** comparar o número de refeições que o cliente disse que faria vs o que realmente fez.
- **Alerta de excesso:** caso o cliente exceda o limite de calorias proposto/imposto por ele mesmo, exibir um **aviso claro e amigável**.
- **Incentivo ao bater meta:** quando uma meta de calorias ou refeições for atingida, exibir um **pop-up de parabéns/felicidades** motivacional e popular (ex: confetes, emoji de troféu, mensagem positiva).
